# REFERENCE.md

本文件用于给数学建模项目提供方法参考、实现路线和报告展示思路。它不是强制工程规范，而是一个“方法路线图”和“实验设计参考库”。

项目对象：2016 年 10 月西安市出租车 GPS 轨迹数据
任务：

* 任务 A：轨迹修复（Trajectory Recovery）
* 任务 B：行程时间估计（Travel Time Estimation, TTE）

---

## 1. 总体判断

这个作业表面上是两个独立任务，但本质上都围绕一个核心问题：

> 如何从出租车轨迹中建模城市出行的时空规律？

任务 A 关注的是 **轨迹空间形状的恢复**：给定稀疏采样点，恢复中间路径。

任务 B 关注的是 **轨迹时间代价的估计**：给定完整路径和出发时间，预测行程耗时。

因此两个任务可以共享很多基础模块：

* 经纬度距离计算
* 轨迹长度计算
* 速度估计
* 方位角变化
* 轨迹曲折度
* 时间特征
* 历史轨迹统计
* 误差可视化

本项目最稳妥的策略不是一开始就追求复杂深度模型，而是形成一条清晰的技术路线：

```text
可靠 baseline
→ 加入时空物理约束
→ 加入历史轨迹统计规律
→ 加入机器学习或模板增强
→ 做消融实验和误差分析
```

课堂评分中“方法创新”和“课堂汇报”占比较高，因此方法不一定要极其复杂，但必须 **能解释、能复现、能对比、能讲清楚为什么有效**。

---

## 2. 两个任务的关系

### 2.1 任务 A 与任务 B 的共性

两个任务都可以从轨迹中抽取这些基础变量：

* 起点位置
* 终点位置
* 总路径长度
* 起终点直线距离
* 曲折度
* 平均速度
* 局部速度
* 出发时间
* 时间段
* 轨迹点数量
* 方向变化

这些变量一方面可以作为任务 B 的特征，另一方面也可以用于任务 A 的轨迹修复约束。

### 2.2 任务 A 可以服务任务 B

如果任务 B 的测试输入是完整路径，但只有出发时间，那么路径形状本身已经给出；任务 A 不直接服务任务 B。

但是任务 A 中实现的这些模块可以复用到任务 B：

* Haversine 距离
* 轨迹总距离
* 方位角变化
* 轨迹平滑
* 异常点检测
* 可视化工具

### 2.3 任务 B 可以反过来解释任务 A

任务 B 中学到的速度模式可以帮助解释任务 A：

* 某些路段速度高，说明轨迹可能经过主干道。
* 高峰期速度低，说明相同距离下时间更长。
* 如果任务 A 恢复出的轨迹隐含速度极不合理，说明恢复可能存在问题。

如果想做一个更统一的展示，可以将两个任务包装为：

> 我们从空间几何、时间节律和历史轨迹三个层面建模城市出租车出行规律，并分别用于轨迹修复和行程时间估计。

## 3. 任务 A：轨迹修复方法参考

任务 A 的难点是：输入点非常稀疏，尤其是 1/16 保留率时，两个已知点之间可能有很长缺口。简单直线插值虽然稳定，但会忽略真实道路的弯曲结构。

因此任务 A 可以按以下层次实现。

---

### 3.1 方法 A0：直接复制输入并检查格式

这是最基础的 sanity check，不用于最终提交。

目的：

* 确认数据读取正确。
* 确认输出格式正确。
* 确认 evaluator 能正常工作。

做法：

* 对已知点保持不变。
* 对未知点暂时填充最近已知点或 0。
* 运行格式检查。

这个阶段的意义不是追求效果，而是避免后续代码写完后才发现格式错误。

---

### 3.2 方法 A1：时间线性插值 baseline

这是必须实现的第一个正式 baseline。

核心思想：

> 在相邻两个已知 GPS 点之间，按照时间比例线性插值经纬度。

公式：

```text
p_t = (1 - alpha) * p_left + alpha * p_right
alpha = (t - t_left) / (t_right - t_left)
```

优点：

* 简单稳定。
* 速度快。
* 适合课堂测试现场运行。
* 在采样间隔较短时通常有不错表现。

缺点：

* 默认车辆在两个已知点之间走直线。
* 对转弯、绕路、复杂道路结构恢复较差。
* 1/16 保留率下误差可能明显增大。

实现重点：

* 经纬度分别插值。
* 必须按 `timestamps` 插值，而不是简单按点序号。
* 已知点坐标必须保持不变。
* 边界缺失要特殊处理：如果开头或结尾存在 NaN，可用最近已知点填充或线性外推，但优先使用最近已知点。

报告中可以这样描述：

> 我们首先构建了一个时间一致的线性插值基线。该方法假设出租车在相邻已知采样点之间以近似匀速运动，并根据真实时间戳对经纬度分别进行插值。该方法为后续速度约束与轨迹模板修正提供了稳定初值。

---

### 3.3 方法 A2：速度约束平滑

在线性插值后，可以检查轨迹中隐含的速度是否合理。

核心思想：

> 真实出租车运动应满足局部速度连续性和最大速度约束。若插值结果导致局部速度突变，则进行平滑修正。

可实现操作：

1. 计算相邻点距离。
2. 根据时间间隔计算速度。
3. 若速度超过阈值，例如 120 km/h，则判定为异常。
4. 对异常区间进行局部平滑或重新插值。
5. 可对未知点使用移动平均，但不要修改已知点。

注意：

* 速度阈值不要设得太低，否则真实快速路段会被错误平滑。
* 平滑强度对 1/8 和 1/16 可以不同。
* 已知点一定不能被平滑改变。

适合做消融：

```text
A1: linear interpolation
A2: linear interpolation + speed smoothing
```

报告可讲创新点：

> 相比纯几何插值，速度约束引入了出租车运动的物理可解释性，能够减少由于稀疏采样导致的不合理跳变。

---

### 3.4 方法 A3：方向变化约束

出租车轨迹通常沿道路行驶，方向变化不会在很短时间内剧烈震荡。

核心思想：

> 对恢复后的轨迹计算局部 bearing，如果连续方向变化异常大，则说明轨迹可能存在不自然弯折。

可实现方式：

* 计算相邻段的 bearing。
* 计算 bearing change。
* 对方向变化过大的未知点进行局部平滑。
* 结合速度阈值共同判断异常。

适合加入报告，但实现时要注意：

* 经纬度坐标下直接算角度可能存在尺度问题。
* 可以先把经纬度近似投影到局部平面坐标。
* 方向约束不一定总能提升指标，但有解释价值。

---

### 3.5 方法 A4：历史轨迹 KNN 模板修正

这是任务 A 中最适合包装成“创新点”的方法。

核心思想：

> 城市出租车轨迹具有重复性。对于一条稀疏轨迹，可以在训练集中寻找起终点相近、方向相似、形状相似的历史轨迹，并利用这些历史轨迹对直线插值结果进行修正。

具体流程：

```text
输入稀疏轨迹
→ 提取已知点、起点、终点、方向、距离
→ 从训练集中筛选候选轨迹
→ 计算候选轨迹与当前轨迹的相似度
→ 选取 top-k 历史轨迹
→ 将历史轨迹重采样到当前时间长度
→ 与线性插值结果融合
→ 输出修复轨迹
```

候选过滤条件：

* 起点距离小于阈值
* 终点距离小于阈值
* 起终点方向差小于阈值
* 路径长度相近
* 轨迹点数相近

相似度可以设计为：

```text
score = w1 * start_distance
      + w2 * end_distance
      + w3 * known_point_alignment_error
      + w4 * direction_difference
      + w5 * length_difference
```

融合方式：

```text
prediction = alpha * linear_prediction + (1 - alpha) * template_prediction
```

其中：

* 1/8 保留率下，已知点较多，可适当提高 template 权重。
* 1/16 保留率下，已知点较少，template 匹配可能不稳定，应更保守。

优点：

* 能恢复非直线路径。
* 能利用训练集中的城市道路先验。
* 很适合作为报告创新点。

风险：

* 计算量较大。
* 如果候选轨迹找错，可能比线性插值更差。
* 需要严格控制运行时间。

建议实现策略：

* 先只取训练集中的部分轨迹做 prototype。
* 再构建起终点网格索引或 KDTree 加速。
* 如果效果不稳定，就作为 advanced method，在最终提交时可以选择是否启用。

报告中可以这样描述：

> 线性插值只能恢复两点之间的直线趋势，而真实出租车轨迹往往受到道路结构约束。为此，我们引入历史轨迹模板修正方法，通过起终点位置、行驶方向和已知点对齐误差检索相似历史轨迹，并将模板轨迹与线性插值结果进行加权融合，从而显式利用城市出行的重复路径模式。

---

### 3.6 方法 A5：基于原始高频数据的监督学习映射

由于 `data_org` 和 `data_ds15` 一一对应，理论上可以利用原始 3 秒数据学习更细粒度运动模式。但任务测试来自 `data_ds15`，所以直接恢复的是 15 秒序列的缺失点。

可考虑方向：

* 用 `data_ds15/train.pkl` 构造模拟缺失任务。
* 随机按 1/8 和 1/16 mask 训练模型。
* 输入稀疏轨迹，输出完整轨迹。

可选模型：

* 简单 MLP
* 1D CNN
* GRU / LSTM
* Transformer encoder

但不建议作为主线，原因：

* 时间成本高。
* 需要处理变长轨迹。
* 小组作业展示中可能难以讲清楚。
* 测试现场稳定性不如插值方法。

更推荐的做法：

* 把监督学习作为“尝试方向”或“未来工作”。
* 主线仍使用可解释的插值 + 约束 + 模板方法。

---

### 3.7 任务 A 最推荐实现路线

建议实际开发顺序：

```text
A1 时间线性插值
→ A2 速度约束平滑
→ A4 历史轨迹模板修正
→ Ablation：比较 A1 / A2 / A4
```

最终提交策略：

* 如果 advanced 方法稳定优于 baseline，则提交 advanced。
* 如果 advanced 在部分场景失效，则保留 baseline 作为 fallback。
* 对 1/8 和 1/16 可以使用不同方法或不同参数。

## 4. 任务 B：行程时间估计方法参考

任务 B 是一个典型的轨迹级回归问题。输入完整坐标序列和出发时间，输出总行程时间。

这个任务的关键不是模型本身多复杂，而是特征设计是否合理。

---

### 4.1 方法 B1：全局平均速度 baseline

核心思想：

> 用训练集估计一个全局平均速度，然后用路径总距离除以平均速度得到行程时间。

公式：

```text
travel_time = total_distance / global_mean_speed
```

优点：

* 极其稳定。
* 可解释。
* 适合作为最低 baseline。

缺点：

* 忽略高峰期、区域差异和路径形状。
* 对短途、拥堵、绕路轨迹误差较大。

报告中作用：

* 作为所有方法的对照基线。
* 说明单纯距离不能完全解释行程时间。

---

### 4.2 方法 B2：分时段速度模型

核心思想：

> 城市交通速度具有明显时间周期性，不同出发时段对应不同平均速度。

可按以下粒度统计速度：

* 按小时：0–23
* 按早高峰 / 晚高峰 / 平峰 / 夜间
* 按工作日 / 周末
* 按小时 + 工作日/周末组合

预测方式：

```text
speed = mean_speed[time_bucket]
travel_time = total_distance / speed
```

优点：

* 仍然可解释。
* 对高峰拥堵有一定建模能力。
* 实现简单。

缺点：

* 忽略空间区域。
* 某些时间桶样本少时不稳定。

改进：

* 如果时间桶样本不足，则回退到全局平均速度。
* 对时间桶速度做平滑。

报告可讲：

> 我们观察到出租车平均速度随出发时间呈现明显日周期变化，因此构建了分时段速度模型，用时段条件速度替代全局平均速度。

---

### 4.3 方法 B3：空间区域速度模型

核心思想：

> 不同城市区域的道路条件和拥堵程度不同，平均速度也不同。

做法：

1. 将西安市经纬度范围划分为网格。
2. 根据起点所在网格统计平均速度。
3. 或根据起点网格 + 终点网格统计平均速度。
4. 预测时查找对应区域速度。

特征：

* start_grid_id
* end_grid_id
* od_pair_id
* start_region_speed
* od_pair_speed

优点：

* 引入城市空间结构。
* 比全局速度更细致。

风险：

* OD pair 稀疏。
* 网格过细会导致很多组合无样本。
* 网格过粗则区分度不足。

推荐做法：

* 先用起点区域速度，不要一开始用 OD pair。
* 网格大小可以从 10x10 或 20x20 开始。
* 对无样本区域回退到时间桶速度或全局速度。

---

### 4.4 方法 B4：轨迹几何特征 + 回归模型

这是任务 B 最推荐的主线方法。

核心思想：

> 行程时间不仅由距离决定，还受路径曲折度、方向变化、出发时间和空间位置共同影响。因此构造多维轨迹特征，并训练回归模型。

推荐特征：

#### 距离类特征

```text
- total_distance
- straight_distance
- distance_ratio = total_distance / straight_distance
- mean_segment_distance
- max_segment_distance
- std_segment_distance
```

#### 轨迹形状特征

```text
- num_points
- tortuosity
- bearing_change_mean
- bearing_change_std
- bounding_box_width
- bounding_box_height
- bounding_box_area
```

#### 时间特征

```text
- departure_hour
- day_of_week
- minute_of_day
- is_peak_hour
- is_weekend
- is_night
```

#### 空间特征

```text
- start_lon
- start_lat
- end_lon
- end_lat
- delta_lon
- delta_lat
- start_grid_x
- start_grid_y
- end_grid_x
- end_grid_y
```

#### 统计速度特征

```text
- global_speed_prediction
- hour_bucket_speed_prediction
- start_region_speed_prediction
```

推荐模型：

```text
- Ridge Regression
- RandomForestRegressor
- GradientBoostingRegressor
- HistGradientBoostingRegressor
- LightGBM / XGBoost if available
```

推荐优先级：

```text
Ridge Regression
→ RandomForestRegressor
→ GradientBoostingRegressor
→ HistGradientBoostingRegressor
```

如果不能安装额外库，直接用 scikit-learn 即可。

评价：

* Ridge 可作为线性模型对照。
* RandomForest 稳定且能处理非线性。
* GradientBoosting 通常对表格数据效果较好。

报告可讲：

> 我们将 TTE 建模为轨迹级回归问题，从距离、形状、时间和空间四个层面构造特征。相比单纯使用路径距离，多层特征能够更充分地刻画城市交通状态和路径复杂度。

---

### 4.5 方法 B5：残差学习

这是很适合包装成创新点的中等复杂度方法。

核心思想：

> 先用可解释规则模型给出一个基础时间估计，再让机器学习模型学习规则模型的误差。

流程：

```text
Step 1: 用 time_bucket_speed_model 得到 baseline_time
Step 2: 计算 residual = true_time - baseline_time
Step 3: 用轨迹特征训练模型预测 residual
Step 4: final_prediction = baseline_time + predicted_residual
```

优点：

* 结合了规则模型的可解释性和 ML 模型的灵活性。
* 模型学习目标更集中。
* 报告中容易讲清楚。

可能比直接预测 travel_time 更稳定。

报告可讲：

> 我们没有让回归模型从零学习行程时间，而是先用时段速度模型构建物理可解释的基础估计，再学习其残差。这种设计将模型重点放在解释拥堵、绕行和局部空间差异造成的偏差上。

---

### 4.6 方法 B6：模型集成

可以将多个模型进行简单加权平均：

```text
final = w1 * time_bucket_speed
      + w2 * random_forest
      + w3 * gradient_boosting
```

权重可以在验证集上网格搜索。

建议：

* 不要搞太复杂的 stacking。
* 简单加权即可。
* 权重搜索范围可以是 0.0 到 1.0，步长 0.1。

适合报告中作为最终优化：

> 为了降低单一模型的不稳定性，我们进一步采用简单集成策略，将规则速度模型与非线性回归模型进行加权融合。

---

### 4.7 任务 B 最推荐实现路线

建议实际开发顺序：

```text
B1 全局速度 baseline
→ B2 分时段速度模型
→ B4 轨迹几何特征 + 回归模型
→ B5 残差学习
→ B6 简单集成
```

最终报告中可以形成清晰对比：

```text
Global Speed
Time Bucket Speed
Feature Regression
Residual Learning
Ensemble
```

这会比只展示一个模型更有数学建模味道。

## 5. 数据探索建议

EDA 是报告和 PPT 的重要部分，不只是辅助代码。

建议至少做以下图表。

### 5.1 轨迹长度分布

统计每条轨迹点数，画直方图。

目的：

* 说明数据轨迹点数范围。
* 解释为什么需要支持变长轨迹。

### 5.2 行程时间分布

从训练集 timestamps 中计算真实 travel time。

目的：

* 展示行程时间大致分布。
* 识别短途和长途样本。

### 5.3 路径距离分布

计算每条轨迹总 Haversine 距离。

目的：

* 说明行程距离与行程时间相关。
* 为 Task B baseline 做铺垫。

### 5.4 平均速度分布

计算：

```text
speed = total_distance / travel_time
```

目的：

* 找异常轨迹。
* 设计速度阈值。
* 支撑 Task A 的速度约束。

### 5.5 不同时段平均速度

按小时统计平均速度。

目的：

* 证明时间特征有意义。

## 8. 可讲的项目创新点

最终报告和 PPT 不建议说“我们用了某某模型”作为唯一创新。应将创新组织成建模思想。

### 8.1 创新点一：时空一致的轨迹建模框架

表述：

> 我们将轨迹数据统一建模为空间几何、时间节律和历史模式的组合，不仅关注点坐标本身，也显式利用时间戳、速度、方向和轨迹形状信息。

适用于两个任务。

### 8.2 创新点二：物理约束下的轨迹修复

表述：

> 在轨迹修复中，我们以时间线性插值作为稳定初值，并进一步引入速度连续性和方向平滑约束，减少稀疏采样下不符合车辆运动规律的恢复结果。

适用于 Task A。

### 8.3 创新点三：历史轨迹模板增强

表述：

> 针对直线插值无法表达道路弯曲的问题，我们检索起终点和行驶方向相似的历史轨迹，并将历史路径形状作为模板先验，与线性插值结果融合。

适用于 Task A。

### 8.4 创新点四：多层特征驱动的 TTE 回归

表述：

> 在行程时间估计中，我们从距离、轨迹形状、出发时间和空间区域四个层面构建特征，使模型能够同时捕捉路径长度、绕行程度、交通时段和城市区域差异。

适用于 Task B。

### 8.5 创新点五：规则估计与残差学习融合

表述：

> 我们先使用可解释的分时段速度模型得到基础时间估计，再训练回归模型学习其残差，从而兼顾物理解释性和非线性拟合能力。

适用于 Task B。

## 12. 可能踩坑与避免方式

### 12.1 经纬度顺序错误

数据格式通常是：

```text
[longitude, latitude]
```

不要写反。

### 12.2 用欧氏距离替代地理距离

最终评估必须使用 Haversine 距离，单位为米。

### 12.3 已知点被修改

Task A 输出中，mask 为 True 的点必须保持原坐标不变。

### 12.4 Task A 评估范围错误

只评估缺失点，不评估已知点。

### 12.5 Task B 单位错误

输出必须是秒，不是分钟。

### 12.6 测试阶段重新训练太慢

Task B 模型应提前训练并保存。课堂测试时只加载模型预测。

### 12.7 高级方法比 baseline 差

这是可能发生的。要保留 fallback，并在实验日志中记录。

### 12.8 文件格式错误

提交文件必须是 list of dict，不是 DataFrame，不是 numpy array。
